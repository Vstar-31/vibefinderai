## 🧠 Entity Detection: A Hidden Bug
The entity hit rate looks like **13.5% miss at @3** — but almost the **entire Travis Scott miss rate (192 cases) is a false failure** . Your catalog stores the artist as `Travi$ Scott` (with a dollar sign) but the NER outputs `Travis Scott`. The fuzzy match fails on that `$` character. This is a **catalog normalization bug**, not an actual retrieval failure. Fix: normalize special characters in artist names during indexing, or use a canonical name registry.

Real entity misses (outside Travis Scott) are mostly on **non-English queries** where artist names from the input get partially mismatched against romanized catalog entries.

## 🔥 Cross-Knob Consistency
Only **35.6% average top-3 overlap** across different knob presets for the same input . That means cranking knobs dramatically reshapes results — which is *intentional* behavior, but 6% of queries have **<10% overlap**, meaning the result set changes almost entirely. This is probably too sensitive for users who expect stable recommendations regardless of knob setting. Consider anchoring the top 1–2 results with a "core match" that persists across knob changes.
